Processing Methodology

Lab 2: NO2 Concentration Maps

Monthly CAMS NO2 data for 2023 was downloaded and processed to compute annual average concentrations. The results were classified into 5 EU standard classes:

ClassRange (μg/m3)Description
1≤10Low
210 - 25Moderate
325 - 40High
440 - 50Very High
5>50Extreme

The Annual Mean Absolute Change (AMAC) was computed as the difference between 2023 and 2021 annual averages: AMAC = avg(2023) - avg(2021).

Lab 3: Land Cover Change

ESA WorldCover 10m data for 2021 and 2023 was used to detect built area changes:

  1. Extract built area (class 5/7) from both years as binary masks
  2. Compute LCC using: Change = 2023_mask x 10 + 2021_mask
  3. Classify into Stable (0-1), Gain (10), Loss (1), NoData (255)
  4. Apply DEFLATE compression with BigTIFF format

Lab 4: Population Exposure & Bivariate Map

Population exposure assessment combines NO2 concentration data with population distribution:

  1. Population density classified into 5 quantile classes
  2. NO2 concentration classified into 5 EU standard classes
  3. Zonal statistics per province: median population class, max pollution class
  4. Bivariate combination: Pol_Class x 10 + Pop_Class (range: 11-55)
  5. Dissolve by pollution class to compute population sums per class

Lab 5: Website & WebGIS

An interactive website was developed to present all project results. The WebGIS component uses OpenLayers with:

  • Three basemaps: OpenStreetMap, CartoDB Positron, Satellite
  • ScaleLine, FullScreen, MousePosition controls
  • GeoJSON vector layers for provinces and bivariate classification
  • Interactive popup with attribute information
  • Layer visibility toggles and basemap switcher

Technical Challenges & Solutions

Large Raster Files

The LCC raster (37K x 27K pixels, 91% NoData) was optimized using BigTIFF with DEFLATE compression, achieving a 106:1 compression ratio (18MB vs 3.8GB uncompressed).

Data Source Updates

2023 CAMS data was used instead of 2020 since the newer data is available. GADM administrative boundaries replaced the unavailable GAUL dataset.

GeoPackage Binary Format

Converting GPKG geometries to GeoJSON required parsing the GeoPackage binary format (flags byte + envelope + standard WKB) to extract coordinate data for the web map.

Raster Alignment

Different spatial resolutions (NO2: 11km, Population: 92m, Land Cover: 10m) required careful resampling and alignment to a common grid for zonal statistics.
